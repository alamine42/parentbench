## Vision

### 10x Check

The current v2.0 vision is: "ParentBench evaluates AI models for child safety."

**10x version:** ParentBench becomes the industry standard for child safety certification — like ESRB for games, but for AI.

| Current Vision | 10x Vision |
|----------------|------------|
| We evaluate models | Providers pay to be evaluated |
| We publish scores | We issue certifications that matter commercially |
| Parents check our site | App stores require our badge |
| 22 models | Every commercial model |
| We run tests | Community contributes test cases, we curate |
| English-only | Global (20+ languages) |
| Point-in-time | Continuous monitoring with alerts |

### Platonic Ideal

Imagine the best engineer with unlimited time and perfect taste built this. A parent would:

1. **Search "is Claude safe for my 12-year-old"** → Google shows ParentBench's answer
2. **See a clear answer** → "Claude 4.5 Sonnet: Certified Safe (Grade A, updated today)"
3. **Drill into details if curious** → Category breakdowns, specific test cases, trend over time
4. **Trust the methodology** → Open-source test cases, reproducible evaluations, expert advisory board
5. **Get notified of changes** → "Claude's safety score dropped 5 points — here's what changed"
6. **Contribute their experience** → "My kid asked X and Claude said Y — is that safe?"

**The feeling:** "I can trust this site like I trust Consumer Reports. They've done the work so I don't have to."

## Implementation Approach

**Approach B: Platform-First Architecture** selected.

Build a real evaluation platform (eval engine, job queue, model adapters, results store) that supports text today and multimodal tomorrow. API-first design. Certification as data layer, not just badges.

### Architecture Components

1. **Eval Engine** — Runs test cases against model APIs, records responses
2. **Job Queue** — Background processing for evaluations (Inngest)
3. **Model Adapters** — Pluggable connectors for each provider's API
4. **Results Store** — Postgres database for historical scores, trends
5. **Notification Service** — Email alerts on score changes (Resend)
6. **API Layer** — Public REST/GraphQL for third parties
7. **Certification Service** — Provider applications, verification workflow

### Infrastructure

- **Hosting:** Vercel (Next.js)
- **Database:** Postgres (Neon or Supabase)
- **Background Jobs:** Inngest
- **Email:** Resend
- **Payments:** Stripe (for certification)
- **Storage:** S3 (for multimodal assets)

### Visual Identity

```
PALETTE
- Primary background: bg-neutral-900
- High-contrast accent for scores
- Status colors: yellow (pending), blue (review), green (approved), red (rejected/revoked)

TYPOGRAPHY
- System fonts (no display fonts)
- High contrast for scores and grades

DIFFERENTIATION
- NOT a marketing site (no testimonials, no "trusted by X")
- Authority through data (scores, test cases, methodology front and center)
- Anxiety reduction (calm, informative language — not alarming)
- Age context pervasive (selection shapes entire experience)

ANTI-PATTERNS (explicitly banned)
- Hero with gradient + 3D illustration
- Grid of feature cards with icons
- Testimonial carousel
- "Start free trial" CTA
- Hamburger menu on desktop
```

### Accessibility Requirements

- Color contrast: 4.5:1 minimum (WCAG AA)
- Touch targets: 44x44px minimum
- Keyboard navigation: All interactive elements focusable
- Screen readers: Scores announced with context
- Focus indicators: Visible outline on all focusables
- ARIA landmarks: banner, navigation, main, contentinfo
